# Vendored: `libcrux-secrets` aeneas-Lean extraction

This directory contains a frozen copy of the aeneas-Lean extraction of
the `libcrux-secrets` Rust crate, vendored here so this repo can build
without requiring a sibling checkout.

## Upstream source

- **Rust crate**: `cryspen/libcrux/crates/utils/secrets` (published to
  crates.io as `libcrux-secrets`, version `0.0.5`).
- **Extraction script**: lives next to the Rust source. In a typical
  local layout it's at
  `~/libcrux-ml-kem-proofs/crates/utils/secrets/hax_aeneas.py`. It
  applies `--exclude core::core_arch::*` to skip SIMD `Scalar` impls
  (NEON / AVX) that aren't relevant to ml-kem and reference types
  outside the rust-core-models pin.

## Layout

```
proofs/aeneas-lean/
├── .gitignore
├── lakefile.toml               # lake config (depends on Hax)
├── lake-manifest.json          # frozen Lake dep manifest
├── lean-toolchain              # leanprover/lean4:v4.28.0-rc1
├── LibcruxSecrets.lean         # root module
└── LibcruxSecrets/
    ├── Missing.lean            # hand-written stubs (none currently)
    └── Extraction/
        └── Funs.lean           # auto-generated by aeneas (~1.8k LOC)
```

## What's in `Funs.lean`

- The `Classify` / `Declassify` / `ClassifyRef` / `DeclassifyRef`
  trait declarations and their `Blanket` (identity) impls.
- The `Select` trait + integer instances (`U8`/`I16`/etc.).
- The `CastOps` trait + 210+ cast-op impl methods for every integer
  pair (`U8 → I16`, `I16 → U8`, …).
- Type aliases: `U8 = u8`, `U16 = u16`, etc.
- Sharing-aware variants (`SharedAT`, `SharedASlice`, `Shared0T`).

Two `ClassifyRefMut for &mut T` impls are block-commented out via a
Python patch in `hax_aeneas.py` — their `T → Result (T × (T → T))`
extracted signature (Aeneas's `&mut` back-fn encoding) doesn't match
the trait's declared `T → Result T` shape. `#[hax_lib::exclude]`
annotations are present in source but aeneas-lean ignores them.

## How to re-vendor

Run the upstream extraction (in the libcrux-secrets crate directory in
the sister repo) and copy the resulting files here, keeping the same
layout. The `.lake/`, `*.llbc`, and `aeneas-error.log` files are
gitignored — only commit `lakefile.toml`, `lake-manifest.json`,
`lean-toolchain`, and the three `.lean` files.

```bash
# In the sister repo:
cd ~/libcrux-ml-kem-proofs/crates/utils/secrets
SKIP_VERSION_CHECK=1 ./hax_aeneas.py

# Then copy outputs here:
SRC=~/libcrux-ml-kem-proofs/crates/utils/secrets/proofs/aeneas-lean
DST=<this-repo>/vendored/libcrux-secrets/proofs/aeneas-lean
cp "$SRC/{lakefile.toml,lake-manifest.json,lean-toolchain,LibcruxSecrets.lean}" "$DST/"
cp "$SRC/LibcruxSecrets/Missing.lean" "$DST/LibcruxSecrets/"
cp "$SRC/LibcruxSecrets/Extraction/Funs.lean" "$DST/LibcruxSecrets/Extraction/"
```
